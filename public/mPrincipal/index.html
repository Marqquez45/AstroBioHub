The Problem We Solve

NASA's Open Access Repository contains 607 critical space biology publications, but this knowledge remains scattered across unstructured PDFs. Researchers spend weeks manually reading papers to answer a single question—time that mission-critical decisions for Mars missions, ISS operations, or planetary protection simply don't allow.

AstroBioHub solves this by using AI and semantic search to transform weeks of literature review into minutes of intelligent synthesis. Our platform processes all 607 papers, identifies patterns across studies, and delivers comprehensive answers backed by verifiable citations—reducing research time by 95%.

What is AstroBioHub?

AstroBioHub is an AI-powered knowledge synthesis platform that transforms NASA's space biology repository into accessible, queryable knowledge. Using advanced natural language processing (PubMedBERT), semantic embeddings, and interactive knowledge graphs, we condense 607 research articles into clear summaries, identify key findings, highlight knowledge gaps, and enable both casual users and researchers to explore space biology efficiently and effectively.

How Does It Work?

AstroBioHub processes 607 NASA publications through automated extraction, creating 4,856 semantic embeddings that enable meaning-based search (not just keyword matching). When you ask "How do organisms survive radiation?", our system finds all relevant concepts—DNA repair, radioresistance, extremophiles—even when papers use different terminology.

The Heart of AstroBioHub

AstroBioHub is an intelligent platform that revolutionizes how scientists and researchers discover, analyze, and synthesize scientific literature. Instead of manually searching for articles one by one, researchers ask questions in natural language and receive deep analyses, cross-study connections, and ready-to-use knowledge synthesis powered by advanced AI and semantic understanding.

How Does It Work? - System's Logical Flow

STEP 1

The Researcher Asks a Question

A scientist has a research question that drives their investigation:

"Which extremophilic organisms can survive in Mars-like conditions?" "How does UV radiation affect bacteria in extreme environments?" "What are the molecular mechanisms of radioresistance in microorganisms?"

The researcher simply types their question into the platform using natural language. The system validates the query structure and accepts optional parameters such as:

  • Date ranges: Filter publications by timeframe
  • Number of results: Control the depth of analysis
  • Document types: Articles, reviews, technical reports
  • Data sources: PubMed, NASA repositories, arXiv
STEP 2

The System Decides the Search Strategy

Not all questions require the same type of analysis. The system has an intelligent processing center that automatically determines which strategy to deploy based on query complexity and requirements:

Four Available Search Strategies:
  • Keyword Search
    For simple and direct searches. Finds exact term matches using traditional information retrieval. Fastest option for specific queries.
  • Hybrid Search
    Combines keyword matches with intelligent ranking algorithms. Balances precision and coverage for moderate complexity questions.
  • Semantic Search
    Understands meaning beyond exact words. Finds related concepts even when expressed differently. Uses embeddings and vector similarity.
  • Deep Analysis with AI
    For complex questions requiring synthesis. Generates summaries, extracts patterns, identifies knowledge gaps, and produces actionable insights.
STEP 3

Intelligent Document Search

This is where the technical magic happens. The system employs two complementary techniques to maximize search effectiveness:

A) Semantic Expansion - Understanding Concepts, Not Just Words
The Problem:
If I search for "extremophilic microorganisms" but a paper uses "thermophilic bacteria," a simple keyword search would completely miss this relevant document, even though they describe the same biological concepts.
The Solution:
The system uses WordNet (a comprehensive linguistic database) to intelligently expand concepts and find semantic relationships:
"extremophile" → EXPANDS TO: thermophile, halophile, acidophile, psychrophile, barophile, radioresistant organism "microorganism" → EXPANDS TO: bacterium, archaea, microbe, prokaryote, unicellular organism "survive" → EXPANDS TO: persist, endure, resist, tolerate, withstand

Result: The search now finds all relevant documents across varied terminology, dramatically increasing recall without sacrificing precision.

B) Intelligent Ranking with BM25 - Sorting by Real Relevance
The Challenge:
Not all articles that mention a search term are equally relevant. We need a mathematical system that intelligently understands:

• A term appearing many times in a short paper is more significant than sparse mentions in a long paper
• Rare and specific terms carry more information than common terms
• Document length must be normalized to avoid bias
The Solution - BM25 Algorithm (Best Matching 25)

Imagine you have 1,000 papers and are searching for "methanogens in ice". BM25 evaluates each document using three intelligent factors:

  • 1. Intelligent Term Frequency (TF)
    • If "methanogens" appears 5 times in a 100-word paper → VERY RELEVANT
    • If it appears 5 times in a 10,000-word paper → LESS RELEVANT
    • Saturation effect: 50 or 100 occurrences both signal central importance (diminishing returns)
  • 2. Inverse Document Frequency (IDF) - Term Rarity
    • "methanogens" appears in only 10 out of 1,000 papers → HIGHLY SPECIFIC (high discriminative power)
    • "the" appears in all 1,000 papers → NO DISCRIMINATIVE VALUE (essentially ignored)
    • Rare terms receive exponentially higher weight
  • 3. Document Length Normalization
    • Long papers naturally contain more words, which doesn't inherently mean higher relevance
    • BM25 adjusts scores relative to average document length
    • Prevents systematic bias toward longer publications

Conceptual Formula (Simplified):

Relevance Score = Term Specificity × Normalized Term Frequency × Length Adjustment
Advantages of the BM25 Approach:
⚡ Fast
Millisecond query times even with thousands of documents
🎯 Accurate
Proven performance in information retrieval benchmarks
💰 Cost-Effective
No GPU required, runs on standard hardware
📈 Scalable
Linear complexity with document count

For complex questions, the system sends the most relevant articles to an AI model (Google Gemini) that acts as an expert research assistant with PhD-level scientific comprehension.

The AI reads each article and systematically extracts key scientific elements:

  • Organisms studied: Species, strains, taxonomic classification

  • Methods used: Experimental protocols, analytical techniques, instruments
  • Quantitative results: Measurements, statistical significance, effect sizes
Example Extraction:

Source Text:
"Strains of Deinococcus radiodurans survived exposure to 15 kGy of gamma radiation at -80°C with 92% cell viability"

• Organism: Deinococcus radiodurans

Phase 1: Intelligent Content Extraction

Mission Objective

Transform 607 heterogeneous scientific publications from NASA's Open Access Repository and PubMed Central into a standardized, machine-readable format optimized for semantic analysis.

Technical Challenge

Scientific publications present significant extraction challenges:

  • Format Inconsistency: Mixed HTML structures across different publishers and journals
  • Content Contamination: Navigation elements, advertisements, and metadata interspersed with scientific content
  • Structural Variability: Non-standardized section naming conventions and hierarchies
  • Encoding Issues: Special characters, mathematical notation, and scientific symbols

Adaptive Extraction Strategy

INPUT EXAMPLE:

https://pubmed.ncbi.nlm.nih.gov/PMC7894523/
        ➜ Full HTML page with navigation, ads, scripts, and content
Three-Layer Identification System:
1. Semantic HTML Parsing
<section id="abstract">, <div class="methods">, <article class="results">
2. Heading Structure Analysis
<h2>Materials and Methods</h2>, <h3>Results</h3>, <h2>Discussion</h2>
3. Pattern Recognition Engine
Regex patterns: "ABSTRACT:", "Introduction", "METHODS:", "RESULTS AND DISCUSSION"

Text Normalization Pipeline

  • HTML Tag Removal: Strip all markup while preserving semantic structure
  • Entity Decoding: Convert HTML entities (&amp; → &, &lt; → <)
  • Whitespace Normalization: Remove excess spaces, tabs, and line breaks
  • Special Character Handling: Preserve scientific notation and mathematical symbols
  • Reference Cleanup: Extract and standardize citation formats

Quality Assurance System

Each extracted paper receives a completeness score based on weighted section presence:

Quality Score = Σ (Sectionweight × Presentbinary)
Section Weight Rationale
Abstract 25% Essential summary of entire study
Methodology 20% Critical for reproducibility and validation
Results 20% Core experimental findings
Introduction 15% Context and research motivation
Discussion 10% Interpretation and implications
Conclusions 5% Summary of key takeaways
References 3% Citation context
Keywords 2% Topic classification

Scoring Examples:

✓ Complete Paper: All 8 sections present → Score = 1.00 (Accepted)
⚠ Partial Paper: Missing Discussion & Conclusions → Score = 0.85 (Accepted)
✗ Incomplete Paper: Only Abstract present → Score = 0.25 (Rejected)
Acceptance Threshold: Quality Score ≥ 0.60

Phase 2: Semantic Vectorization & Retrieval

Mission Objective

Transform human-readable scientific text into high-dimensional numerical vectors that capture semantic meaning, enabling context-aware similarity searches across the entire corpus.

Understanding Semantic Embeddings

Conceptual Framework:

Embeddings create a mathematical "meaning space" where semantically similar concepts cluster together. Think of it as a 768-dimensional coordinate system where each axis represents a learned linguistic or scientific feature.

SEMANTIC PROXIMITY EXAMPLES:

        Close in embedding space:
        • "bacteria" ←→ "microorganism" ←→ "prokaryote"
        • "microgravity" ←→ "weightlessness" ←→ "zero-g"
        • "DNA repair" ←→ "genome stability" ←→ "mutation prevention"

        Distant in embedding space:
        • "bacteria" ←--far--→ "spacecraft propulsion"
        • "gene expression" ←--far--→ "orbital mechanics"

Each text segment is converted into a 768-dimensional vector using PubMedBERT, capturing nuanced scientific meaning beyond simple keyword matching.

Model Selection: PubMedBERT + MS MARCO

Component Purpose Benefit
PubMedBERT Pre-trained on 14M+ biomedical abstracts Deep understanding of scientific terminology, protein names, biological processes
MS MARCO Fine-tuning Optimized on 1M+ query-document pairs Superior information retrieval performance, understands search intent

COMPARATIVE ADVANTAGE:

Generic Language Model (e.g., BERT-base):
        Query: "thermophilic archaea"
        ❌ May retrieve: thermal insulation, architectural heating systems
        ❌ Lacks domain-specific context

        PubMedBERT + MS MARCO:
        Query: "thermophilic archaea"
        ✓ Retrieves: Pyrococcus furiosus, Sulfolobus solfataricus
        ✓ Understands: extremophiles, hyperthermophiles, archaeal biology
        ✓ Semantic expansion: heat-resistant microorganisms, thermal adaptation

BERT Architecture: Self-Attention Mechanism

BERT's core innovation is self-attention, which allows each word to understand its context by attending to all other words in the sequence.

EXAMPLE SENTENCE:

"Bacteria survive extreme radiation"

Context understanding through attention:

• "Bacteria" attends to → "survive" (action) + "radiation" (stressor)
        • "survive" attends to → "Bacteria" (subject) + "extreme" (intensity)
        • "extreme" attends to → "radiation" (what is extreme)
        • "radiation" attends to → "Bacteria" (affected entity) + "survive" (outcome)

Self-Attention Mathematical Formula:

Attention(Q, K, V) = softmax(QKT / √dk) × V
Parameter Description Dimension
Q (Query) "What am I looking for?" n × 64
K (Key) "What information can I provide?" n × 64
V (Value) "The actual information content" n × 64
dk Scaling factor (prevents vanishing gradients) 64

Similarity Measurement: Cosine Similarity

Mathematical Definition:

cosine_similarity(A, B) = (A · B) / (||A|| × ||B||)

Interpretation:

  • 1.0: Identical meaning (perfect match)
  • 0.7 - 0.9: Highly related (strong semantic similarity)
  • 0.3 - 0.7: Moderately related
  • 0.0: Completely unrelated
  • -1.0: Opposite meaning (rarely occurs in practice)
